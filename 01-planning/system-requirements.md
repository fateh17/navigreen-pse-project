## Functional and non-functional requirements 

# Functional Requirements 

1 – User Login

The system should allow users to log in using their email address and password.

2 – Trip Input

The system shall allow users to enter:
	•	Starting location
	•	Destination
	

3 – Transport Selection

The system allows users to select a transportation method (Car, Bus, Bicycle, Walk, Motorcycle).

4 – Carbon Emission Calculation

The system shall calculate and display the estimated carbon emission based on:
	•	Distance
	•	Selected transport type

5 – Display Carbon Savings

The system should display the amount of carbon saved when an eco-friendly transport option is selected.

6 – Weekly Goal Setting

The system shall allow users to set weekly carbon reduction goals.

7 – Reward System

The system should rovide a reward when users successfully complete their carbon-saving goal.

 8– Environmental News Page

The system shall display updated environmental news, eco-tips, and climate-related information.



# Non-Functional Requirements
 

1 – Usability

The application must have a simple and user-friendly interface that is easy to navigate.

2 – Performance

The system shall display carbon calculation results within 2 seconds after the user clicks


3 – Consistency

The system shall maintain consistent colors, fonts, and layout across all screens.

4 – Accessibility

The text and buttons should eclearly visible with readable font sizes and sufficient contrast.

5 – Reliability

The system must provide accurate emission estimates based on predefined carbon emission values.

6 – Security

User login information must be securely handled and protected from unauthorized access.
