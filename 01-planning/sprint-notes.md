# Week 1 Notes

Duration: 16 Feb – 22 Feb  

Goals: 
- assign roles
- create trello board
- create user stories
- set up repository
- write product vision


### What went well
- Clear task delegation
- Well-defined requirements in each Trello card
- Efficient tracking of progress using the To Do > Doing > Done workflow
  
### What didn't go well
- Occasional misunderstandings that required clarification in the Google Doc
- Some minor delays in moving cards, which did not always reflect real-time progress

### Improvements for the next sprint
-	Review each task to ensure quality before moving it to “Done.”
-	Set progress checks before the final deadline to better manage time.
-	Update Trello cards in real time to ensure the board accurately reflects progress.
